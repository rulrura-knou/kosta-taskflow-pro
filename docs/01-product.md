# 01-product.md — 제품 기획서

## 목표

> **"지금 누가 뭐해?"** 라는 질문이 사라지게 한다.

팀 업무를 실시간으로 시각화하여, 별도 보고 없이도 팀 전체의 진행 상황을 한눈에 파악할 수 있게 한다.

---

## 페르소나

| 항목 | 내용 |
|---|---|
| 대상 | 10명 규모 스타트업의 팀 리더 |
| 연령대 | 30~40대 |
| 상황 | 매일 아침 슬랙에서 "지금 어디까지 했어요?"를 반복하고 있다 |
| 목표 | 팀원 각자의 업무 상태를 물어보지 않고도 파악하고 싶다 |
| 불편 | 엑셀 공유, 구두 보고, 미팅으로 상태 파악에 시간을 낭비하고 있다 |

---

## MVP 범위

### CRUD 4종 — 모두 화면에서 동작

| 기능 | 설명 |
|---|---|
| 추가 (Create) | 태스크 제목, 상태, 마감시각을 입력해 생성 |
| 목록 (Read) | 전체 태스크를 상태별로 분류하여 표시 |
| 수정 (Update) | 제목, 상태, 마감시각을 인라인 또는 모달에서 수정 |
| 삭제 (Delete) | 태스크 삭제, 확인 절차 포함 |

### 상태 분류

- `todo` — 할 일
- `in_progress` — 진행 중
- `done` — 완료

### 마감시각 (`due_at`)

- 날짜만이 아니라 **시간까지** 지정한다.
- 표시 형식 예시: `2026-05-12 18:00`
- DB 저장 형식: ISO 8601 (`2026-05-12T18:00:00`)

### 테마

- 라이트 / 다크 테마 토글
- 선택값은 `localStorage`에 저장하여 새로고침 후에도 유지

### 반응형

- 모바일 최소 기준: **360px**
- 레이아웃이 깨지지 않아야 한다

## UI 톤 & 무드

Mac OS 스타일을 기준으로 한다.

| 요소 | 방향 |
|---|---|
| 모서리 | 둥근 모서리 (border-radius 넉넉히) |
| 그림자 | 부드러운 그림자 (box-shadow, 강하지 않게) |
| 카드 | 반투명 카드 (backdrop-filter 또는 rgba 배경) |
| 폰트 | 시스템 폰트 (`-apple-system, BlinkMacSystemFont, sans-serif`) |
| 전체 느낌 | 가볍고 깔끔함. 정보 밀도보다 여백과 가독성 우선 |

## 확장 범위 (MVP 이후)

MVP 완료 후 다음 단계에서 추가한다. 현재 설계에서 구조를 열어 두되 구현은 보류한다.

| 기능 | 비고 |
|---|---|
| JWT 로그인 | 인증/인가 도입 |
| 팀 기능 | 멤버 초대, 역할 관리 |
| Kanban 보드 | 드래그앤드롭 상태 변경 |
| 채팅 | 태스크 단위 댓글 또는 팀 채팅 |
| CI/CD | 자동 배포 파이프라인 |

## MVP 범위 외 (구현하지 않음)

다음 항목은 MVP에서 명시적으로 제외한다.

| 제외 항목 | 이유 |
|---|---|
| 외부 결제 | 과금 모델 미정 |
| 네이티브 앱 | 웹 반응형으로 대체 |
| WebRTC | 영상/음성 통신 불필요 |
| 외부 캘린더 연동 | Google Calendar 등 OAuth 범위 초과 |
| 파일 업로드 | 스토리지 설계 미포함 |
| WebSocket | MVP는 **폴링** 방식으로 처리. 실시간 연결은 확장 단계에서 도입 |

## 성공 기준

MVP가 아래 조건을 모두 충족하면 완료로 간주한다.

| 기준 | 조건 |
|---|---|
| 데이터 유지 | 새로고침 후에도 태스크 목록이 유지된다 |
| 반응형 | 360px 화면에서 레이아웃이 깨지지 않는다 |
| 성능 | CRUD 4종 API 응답시간 200ms 이하 |
| 기능 완결 | CRUD 4종이 모두 화면에서 동작한다 |
| 테마 | 라이트/다크 토글이 작동하고 새로고침 후에도 유지된다 |
